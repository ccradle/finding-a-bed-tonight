## For Faith Community Boards, Volunteer Shelter Operators, and Non-Technical Decision-Makers

### The Short Answer

**The software is free. Hosting it costs about $15-30 per month.**

Think of it like a free building plan. The plan is free — anyone can use it. But you still need land and materials. For FABT, the "land and materials" is a small cloud server.

### What "Free" Means

| What You Get | Cost |
|---|---|
| The software itself | **$0 — forever** |
| Updates and new features | **$0** |
| Number of users | **Unlimited — no per-seat fees** |
| Annual license renewal | **None — there is no license to renew** |

FABT is released under the Apache License 2.0, which means:
- You can use it for free, forever
- There are no hidden fees, no upsells, no "free tier that expires"
- No company can take it away or raise the price
- Your data belongs to you, not to a software vendor

### What You Do Pay For

| What You Pay For | Typical Cost | What It Is |
|---|---|---|
| Cloud server hosting | $15-30/month | A small computer in the cloud that runs the app and database |
| Domain name (optional) | $12/year | A web address like "beds.yourorg.org" |
| SSL certificate | $0 | Encrypted connection — included free with Let's Encrypt |

**That's it.** For a rural CoC or a network of faith community shelters, the total cost is comparable to a Netflix or Spotify subscription.

### What "Free" Does NOT Include

These are real costs, but they're the same costs you'd have with any software:

- **Someone to set it up** — Initial deployment takes a few hours of technical work. This can be done by a volunteer with IT skills, a local Code for America brigade, or a paid IT consultant.
- **Someone to keep it running** — Software occasionally needs updates. For a small deployment, this is a few hours per quarter.
- **Someone to call when it breaks** — The project has documentation and a GitHub community, but there is no 24/7 phone line. See the Support Model document for more detail.

### Can We Participate With Just Bed Counts?

**Yes.** You can use FABT for bed count updates only — no reservations, no DV referrals, no surge events. A coordinator logs in, updates the occupied count, and logs out. That's it. The system is designed so each organization can participate at the level that works for them.

### What Happens If We Stop Using It?

Nothing bad. Your data stays on your server. You can export shelter inventory as CSV. There is no contract to cancel, no early termination fee, and no vendor to notify. If your shelter is seasonal (like White Flag nights), the system sits idle during off-season at no cost beyond the hosting fee.

### What Happens If the Project Disappears?

The software is permanently available on GitHub under an irrevocable open-source license. Even if the project team stops working on it, your deployment continues to run, and any IT professional can maintain it. This is the same assurance you have with WordPress, Firefox, or any open-source software.

### Who Pays for the $15-30/Month?

Options that other organizations have used:
- The CoC includes it in their operating budget
- A participating city or county covers hosting as part of IT infrastructure
- A foundation grant covers the first year
- Multiple organizations split the cost (at $15-30/month, splitting among 5 organizations is $3-6 each)

### Questions to Ask Your Board

1. "Are we currently spending staff time making phone calls to find beds?" (If yes, FABT replaces that time)
2. "Can a volunteer with IT skills set this up?" (Usually yes — the deployment guide is step-by-step)
3. "Can we try it without committing?" (Yes — set up a test environment for $0 using the free development tools)
